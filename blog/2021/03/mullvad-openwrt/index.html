DRAFT: Mullvad：openWRT

OpenWRTのトラベルルーターにWireguardの設定をする。 サポートページに接続方法が書いてあります。

DNSをNextDNSを使いたいことからちょっと問題があったりしましたが、 問題なく動いているようです。

えた方が良いのかちょっとこの辺がわからず。

NextDNSにより、 今まで使っていたAdblockは不要に、 Shubby経由でNextDNSに繋げます。

とりあえず動作はしている感じです。

Wireguard

  • https://mullvad.net/en/help/running-wireguard-router/

Initial configuration

SSH Access : set the interface to LAN. Click the Save & Apply button.

Install wireguard

Install Packages

root@OpenWrt:~$ opkg install wireguard luci-proto-wireguard luci-app-wireguard

鍵を作成します

root@OpenWrt:~$ wg genkey | tee privatekey | wg pubkey > publickey

Wireguard設定

WireGuard interfaceの追加

In the main menu,

select Network- Interfaces and then click on “Add new interface”.

Name the interface WGINTERFACE and select Protocol WireGuard VPN and press submit in order to start configure the new interface.

Configure WGInterface settings

接続先をサーバーリストより選びIPアドレスと、公開鍵をコピーしておきます。 コマンドを打ってIPアドレスを所得します。

root@OpenWrt:~$ curl https://api.mullvad.net/wg/ -d account=##ACCOUNTID## –data-urlencode pubkey=vsh2XXXXXXXXXXX

  • IP Addresses – replace 10.99.0.5 with the IP address you received from Mullvad
  • Private Key – 作成した秘密鍵（privatekey）
  • Public Key – 接続先Mullvad WireGuard サーバーの公開鍵
  • Route Allowed IPs - チェクマークにチェック
  • Endpoint Host (Peers)– 接続先Mullvad WireGuard サーバーのIPアドレス
  • Endpoint Port - 51820
  • Allowed IPs - 0.0.0.0/0

List of servers: https://mullvad.net/servers

Add new firewall zone

Scroll down to Zones. Create a new zone and set it up as shown below. We named ours “WGZONE”. Click on Edit the lan zone and set “Allow forward to destination zones” to WGZONE:WGINTERFACE" and also check that your settings matches the picture below.

DHCP and DNS settings

root@tortas:~# vi /etc/config/stubby

config resolver option address ’ ' option tls_auth_name ‘mullvad.net’

Change DNS on LAN Interface

Navigate to Network→Interfaces→LAN and make the following changes:

  • IPv4 address – change this to “192.168.99.1” (this ensures that it won’t conflict with our other routers commonly running on 192.168.0.1 or 192.168.1.1) . Keep in mind after this you will access the OpenWRT device on 192.168.99.1

  • DHCP-Options – set this to “6,10.64.0.1”. (If you’d like to learn more about DNS, check out our guide on DNS leaks.)
