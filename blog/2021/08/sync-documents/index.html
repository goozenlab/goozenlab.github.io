DRAFT: 書類の同期

大まかにzfsにてバックアップ・同期を行なっています。 マックではタイムマシーンがありますが、 頑なに拒否してます。（別に嫌な思い出があるわけはない）

定期的にクリーンインストールするというのが良いものだとの思い込みがあるので、 必要なものバックアップしておくという古典的な方法をとってます。

個人的に複数箇所にバックアップを取っておくというのが良いと思っていて、 ローカルディスク・バックアップディスク・クラウド の三箇所であれば安心できるかなと、、

バックアップディスクはZFSを利用しミラーリングで２つ３つのディスクを使ってミラーリングしています。

ローカル：ミラーリングディスク：クラウド保存

AFP：Documents ZFS: LocalPartition Backup ZFS: LocalPartion Sync

で各所に分散していればまぁ、kそこまで大事なファイルがあるわけではありませんが、 大事には至らないのではとはと思ってます。 （個人的に３箇所もしくは３つのバックアップ、３つ以上管理するのは）

ミラーリングのディスクは実家（一時帰国時）・ノマド用（外部SSDと端末） ・タイ（海外滞在時） に置いてあって、自分の中でどれがどの状況なのかがわかります。

最近はHDDの容量が大きいので

しかしマック上のDocumentsフォルダー内のファイルに至っては手をつけていなかったので、 同期・バックアップを行うことにしました。

方法

色々と考えて末（でもないかな。。）以下の形にしました。

  • fswatch: 更新ファイルの監視
  • unison: フォルダーの同期
  • rsync: バックアップ

fswatchでDocuments内とSyncthingで同期をしているフォルダーをチェックし、 更新などがあった場合に同期とバックアップを同時に行います。

fswatch: フォルダーの監視

フォルダーを監視する。 ファイルが追加されたり、更新された時に

fswatch

ディレクトリを監視して任意のコマンドを実行する

fswatchコマンドを使うと指定したディレクトリで変更のあったファイルを標準出力に吐いてくれる．fswatchコマンドはkqueueやionotifyを利用したツールで，MAC OS Xを含むUNIX上や，Linux上で動作する．

emcrisostomo/fswatch · GitHub

# Homebrew
$ brew install fswatch

これを利用すると，指定したディレクトリで変更があった際に自動でbuildする，と言ったようなことが比較的手軽に実現できてお得．

たとえばこういう感じ

fswatch <監視したいディレクトリ> | xargs -n1 -I{} <任意のコマンド> ファイル名を引数にして渡したいときはxargsの-l{}を消したりすれば良い．詳しくはxargsのマニュアルを見てくれ!!!!

Emacsでファイルとか編集すると一時ファイルとかできるから，現実的には以下のようになると思う．

fswatch <監視したいディレクトリ> | grep -v –line-buffered ‘# ‘ | xargs -n1 -I{} <任意のコマンド>

fswatchを使うと楽っぽい。

$ brew install fswatch
$ fswatch . | xargs -n1 -I{} gofmt -w {}

gofmt -w だと、stdoutに出力せずに直接対象ファイルに書き込まれる。

$ fswatch -o /path/to/src1 /path/to/src2 | xargs -I{} unison -batch sync

rsyncでバックアップ

1.rsyncを走らせてバックアップ 2. unisonを走らせてsyncthingにて同期しているフォルダーを更新

手元にあるNoodleでも実行は可能ですが、 フリーな環境で仕上げてみました。

Rsync

バックアップは一方通行なのでrsyncで行います。 ~/Documets -> /Volumes/backup/Documents (mirror zfs)

Unisonでフォルダーの同期

同期先のフォルダーはローカルだけではなくて、アンドロイドとも同期をしているのでそちらの更新もあるので、

vi ~/.unison/sync_documents.prf

# sync_documents.prf
# 書類を同期＋zthumbへバックアップ

# タイムスタンプをコピーする
times = true

# 新しいファイルを優先
prefer = newer

auto = true
fastcheck = true

# 動作確認が取れたらコメントを外してよし。
#silent = true

# ルート・ディレクトリの指定。同期時に最上位となるディレクトリを設定する。
root=/Users/jin/Documents
root=/Volumes/Sync/Documents

# 同期したいディレクトリやファイル名を指定する：空白は全て
path = Travel
path = Finance
path = 

# 除外ファイルリスト
include ignore_filelist_macos

unisonによる同期を実行したい場合は-batch オプションを付けて実行すると非対話的に同期が実行されます。

必要なコマンドは

fswatch -o /Volumes/Work | xargs -n1 -I{} echo changed

fswatch ~/Documents “echo changed”

rsync -ahvn /Users/jin/Documents/ /Volumes/Backup/Documents

unison -batch sync_documents unison sync_documents

$ fswatch -o /Users/jin/Documents /Volumes/Sync/Documents | xargs -I{} unison -batch sync

Launched

ログイン時にスクリプトを走らせます。 Launchedサービスを使い行います。

サービスを登録します。

Label goozenlab.sync.documents ProgramArguments /usr/local/bin/fswatch /Users/jin/Documents unison -batch sync_documents/ RunAtLoad KeepAlive UserName jin 
<key>KeepAlive</key>
<dict>
  <key>Crashed</key>
  <true/>
  <key>SuccessfulExit</key>
  <false/>
</dict>
<key>ProcessType</key>
<string>Background</string>

rsync ファイルを オプションはあたくさんありますが、

RunAtLoad: 起動時に実行 KeepAlive: 自動で再起動を試みる

そしてロードします。

$ sudo launchctl load /Library/LaunchDaemons/com.consul.plist ちなみに停止するには unload してやればいいです

$ sudo launchctl unload /Library/LaunchDaemons/com.consul.plist
