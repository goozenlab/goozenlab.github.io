Standard Note

Simplenoteともう一つ。ということで、Standardnotesをインストールしました。

こちらのアプリはサーバーも抱き合わせのオープンソースプロジェクトですので セルフ・ホストも可能のようです。ドッカーで一発構築できるようです。 ラズベリーパイで軽快に動かせたらよいのだけどね。。

全てのプラットフォームに移植されていて、LinuxではAppImageで提供されています。

アプリはライブラリーを含んだ配布方式のAppImageです。実行権を与えてダブルクリック。。 で動かない。。

ターミナルで立ち上げるとエラーが出てくる。。

jin@debian:/opt$ ./Standard-Notes-3.0.25.AppImage [3563:0116/185840.711069:FATAL:setuid_sandbox_host.cc(157)] The SUID sandbox helper binary was found, but is not configured correctly. Rather than run without sandboxing I’m aborting now. You need to make sure that /tmp/.mount_Standa3whW6B/chrome-sandbox is owned by root and has mode 4755. Trace/breakpoint trap

検索後解決方法が見つかる。–no-sandboxのオプションを追加。。でうまく動きました。。 でも毎回ターミナルで立ち上げるのもな、、

jin@debian:/opt$ ./Standard-Notes-3.0.25.AppImage –no-sandbox Extensions server started at http://localhost:45653 Unable to read JSON file /home/jin-debian/.config/Standard Notes/Updates/settings.json Checking for updates… [18:56:58:0161] [info] Checking for update [18:56:58:0430] [info] Generated new staging user ID: 74c5e1cd-f242-5a86-a4c7-3890b03245ea Syncing components: 1 Syncing components: 1 Syncing components: 1 Finished checking for updates. Latest version: 3.0.25 Current version: 3.0.25 Data backup succesfully saved: 2020-01-16T11-57-01.638Z.txt [18:57:02:0256] [info] Update for version 3.0.25 is not available (latest version: 3.0.25, downgrade is disallowed). Syncing components: 1 Error: Object has been destroyed at /tmp/.mount_StandakVj231/resources/electron.asar/browser/api/web-contents.js:245:33 at processTicksAndRejections (internal/process/task_queues.js:79:9)

何が問題なのかはちょっとわからないけれど、オプションを追加せずに起動する方法もあるみたいです。

  1. Enable unprivileged access to CLONE_NEWUSER in your kernel. Some kernels support changing this with sysctl kernel.unprivileged_userns_clone=1.
  2. Disable sandboxing entirely by launching with –no-sandbox. Adding this argument from JS is unfortunately insufficient, as the GPU process is launched before the main process JS is run.

/etc/sysctl.d/00-local-userns.conf

with the following contents:

kernel.unprivileged_userns_clone=1

でファイルをダブルクリックで実行できます。 アイコンの登録もできんのかな。。

Visual Studioも軽快に動いているし、Moscもまぁそんなに悪くない。

どんどん、ただ、こちらLinuxは複雑化して、いろいろ使いまわしてます。

今の所、デビアンの出来が非常に良いのでメインになりつつあり、 Manjaroも悪くないのですが、KDE Neonがかーなりよく作られてます。 デフォルトの設定がBashとかVimとかの、初心者向けによく作られてます。
