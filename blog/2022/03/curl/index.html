ダウンロードの再開

実家回線は光なので、データーをダウンロードが途中で切れても、 もう一度やり直してもそこまで気にはなりません。 ただし回線状況のあまり良くないところで、ダウンロードの最後の方で回線が切れてのやり直しはちょっとイラっときます。

例えば、Xcodeをダウンロード（だいたい５GB）を３日ぐらいかけてダウンロードをしていて、 あと少しで終わるところで、レジュームが効かなくなりやり直し、、 という経験が計画停電のあるネパールでほそーーい回線で起こったことがある

そこで、少し大きめのファイルをダウンロードするときには 中断後再開できるようにしておきます。

たまーにしか使わないので、度々調べては忘れてしまうのでメモに残しておく。

まぁ、簡単なことで、オプションをつければよろしい。

海外のちょっと回線が悪いところなどで、 大きめのファイルを落とすときにつかう。

$ curl -O  http://iso.mirrors.ustc.edu.cn/kde-application/neon/images/user/20200102-1116/neon-user-20200102-1116.iso
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed
  0 1477M    0 3175k    0     0   120k      0  3:28:46  0:00:26  3:28:20  153k

中断後再開

$ curl -L -O -C - http://iso.mirrors.ustc.edu.cn/kde-application/neon/images/user/20200102-1116/neon-user-20200102-1116.iso
** Resuming transfer from byte position 3330048
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed

ギットハブからのダウンロード？からだとうまくいかなかったりしますが、 ほとんどの状況で使えますのでご参考のほど。
